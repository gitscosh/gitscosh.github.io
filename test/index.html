Boolean

Boolean operators aren’t just evaluated from left to right. Just like with arithmetic operators, there’s an order of operations for boolean operators:

  1. not is evaluated first;
  2. and is evaluated next;
  3. or is evaluated last.

For example, True or not False and False returns True. If this isn’t clear, look at the Hint.

Parentheses () ensure your expressions are evaluated in the order you want. Anything in parentheses is evaluated as its own unit.

Integers

  • decimal: 10
  • binary: 0b10
  • octal: 0o10
  • hex: 0x10

:boom: Note

  • Rounding is always toward zero.

Division operator to return an int rather than a float: //

Strings

  • sequences of unicode code points

Multiline strings can be denoted with triple quotes at the beginning. You can also insert the newline yourself with \\n

  • \\ is an escape sequence

:bulb: Hint

  • Rather than escape a lot in a string, such as the path to a file, you can use r (raw) at the beginning.
  • path = r’C:\Users\Merlin\Documents\Spells’

Bytes

  • sequences of bytes
  • used for raw binary data and fixed width, single byte character encodings such as ASCII
  • support most of the same operations as strings, such as indexing and splitting

:page_with_curl: Bytes Literals

  • b'data'
  • b"data"
  d = "some bytes"
  d.split()
  >>> [b"some", b"bytes"]
  • To convert between bytes and strings, we must know the encoding of the byte sequence used to represent the string’s unicode code points as bytes
  • string.encode('utf-8')
  • string.decode('utf-8')

:warning: Important!

  • files and network resources, such as HTTP responses, are transmitted as byte streams

For Loops

  colors = {"crimson": 0xdc143c, "coral": 0xff7f50, "teal": 0x008080}
  for color in colors:
      print(color, colors[color])

Putting It All Together

  from urllib.request import urlopen
  with urlopen("http://sixty-north.com/c/t.txt") as story:
      story_words = []
      for line in story:
          line_words = line.decode("utf-8").split.()
          # above line needed to return a list of strings, rather than a list of byte objects
          for word in line_words:
              story_words.append(word)
